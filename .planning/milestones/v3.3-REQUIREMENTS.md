## v3.3 Requirements

Requirements for this milestone were defined inline in the phase plan files. Extracted here for archival.

### MCP-TOOL — Agent-Facing MCP Tools

- [x] **MCP-TOOL-01**: Agent can call `mcp_add(name, url)` — HTTP MCP server entry appears in `.claude.json` with `type: http`; `https://` URL required
- [x] **MCP-TOOL-02**: Agent calling `mcp_remove("rightmemory")` receives an error; `.claude.json` is unchanged (rightmemory is a protected server)
- [x] **MCP-TOOL-03**: Agent can call `mcp_list()` — receives JSON array with `name/url/auth/source/kind` fields; no token values exposed
- [x] **MCP-TOOL-04**: Agent can call `mcp_auth(server_name)` — receives the AS `authorization_endpoint` URL (RFC 9728/8414 discovery); no PKCE/DCR performed; agent instructed to use Telegram bot for full auth flow
- [x] **MCP-TOOL-05**: All four tools (`mcp_add`, `mcp_remove`, `mcp_list`, `mcp_auth`) appear in the MCP server's `tools/list` response via tool_router registration

### MCP-NF — Non-Functional

- [x] **MCP-NF-01**: `mcp_list` returns no token or credential values — satisfied by library design (`mcp_auth_status` returns auth state only)
- [x] **MCP-NF-02**: `MemoryServer::new()` accepts `agent_dir: PathBuf` and `rightclaw_home: PathBuf`; `generate_mcp_config()` injects `RC_RIGHTCLAW_HOME` into rightmemory env section of `.mcp.json`
