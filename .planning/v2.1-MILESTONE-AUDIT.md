## Tech Debt

### Phase 9 — git doctor check placement

The plan 09-02 description says "git Warn doctor check" but the implementation puts the warning in `verify_dependencies()` (called by `cmd_up`), not in `run_doctor()`. AENV-01 only requires git init on `up`, so this is not a requirement violation. However it means the doctor output doesn't reflect git absence.

**Fix:** Add `check_binary("git", None)` with `CheckStatus::Warn` to `run_doctor()` in `doctor.rs`, after the existing bwrap/socat block, Linux-gate-free (mirrors TOOL-02 pattern).

### Phase 8 — unautomatable live tests

Two ignored scaffold tests in `home_isolation.rs` require live credentials and a running agent process. These are correctly marked `#[ignore]` and documented. Not blocking.
